# 미니 프로젝트 — 자기소개 페이지 리뉴얼

## ① 자기소개

교육학을 8년 전공하고 UX 기획·고객 분석 업무를 15년 해온 이지용입니다. 빠르게 시도하고 반복해서 개선하는 실행력, 반복 업무를 간소화·자동화하는 실무 효율화, 그리고 협업을 통해 함께 성장하는 것을 중요하게 생각합니다. 취미는 서핑과 스쿠버다이빙, 그리고 책을 읽고 이야기 나누는 것을 좋아합니다.

이번 부트캠프에서는 하루 1개 이상 직접 코딩하기, 주 단위 데이터 시각화 게시, 월 단위 MVP 베타 오픈, 교육 기간 내 협업 파이프라인 가동을 목표로 잡았습니다.

## ② HTML에서 신경 쓴 부분

- Day1에서는 `ul/li`와 `h1~h3`만으로 구조를 짰던 부분을, `header / nav / main / section / footer` 시맨틱 태그로 재구성했습니다.
- `main`은 페이지 전체에서 한 번만 사용하고, 각 내용 블록(소개/취미/목표/정보표)을 의미 단위로 `section`으로 나눴습니다.
- `h1`은 페이지 대표 제목(자기소개) 하나로 정리하고, 각 section 제목은 `h2`로 통일해서 heading 구조가 계단식으로 이어지도록 했습니다.
- 기존 mindset.html에 `head` 안에 `body`가 들어가 있던 오류를 발견해서 `head`를 제대로 닫아주고 문서 구조를 다시 잡았습니다.
- 이미지에는 전부 `alt` 속성을 넣어서 대체 텍스트를 확인했습니다.

## ③ CSS에서 신경 쓴 부분

- 전체 요소에 `box-sizing: border-box`를 적용해서 padding/border를 포함한 크기 계산이 되도록 했습니다.
- 프로필 이미지, 학습 목표 카드, 표 셀 등에 `padding / border / margin`을 직접 지정해서 박스 모델을 눈으로 확인할 수 있게 만들었습니다.
- 태그 선택자, id 선택자, class 선택자, 후손 선택자(`.site-header a`), 자식 선택자(`.navbar-nav > .nav-item`), 속성 선택자(`a[href$=".html"]`)를 각각 최소 1번 이상 의도적으로 사용해봤습니다.
- 네비게이션 링크에는 `::after` 가상요소로 밑줄이 채워지는 hover 효과를, 프로필 이미지와 카드에는 `:hover` 가상클래스로 확대/그림자 효과를 넣었습니다.
- 학습 목표 영역은 Bootstrap grid를 쓰지 않고 `display: flex; flex-wrap: wrap; gap: 20px`를 직접 작성해서 Flexbox 개념을 스스로 구현해봤습니다.
- 서핑·스쿠버다이빙이라는 제 취미에서 착안해서 바다색(딥블루~스카이블루) 팔레트를 커스텀 CSS 변수(`--color-deep`, `--color-ocean` 등)로 만들어 적용했습니다.

## ④ Bootstrap은 어디에 사용했나요?

- CDN 방식으로 Bootstrap 5를 `head`에 연결했습니다.
- `navbar`, `navbar-toggler`로 반응형 상단 메뉴를 구성했습니다.
- `container / row / col` 그리드로 자기소개 영역(이미지+텍스트)을 2단 배치했습니다.
- 취미/좋아하는 것 영역은 `row-cols-1 row-cols-sm-2 row-cols-md-3`로 화면 크기에 따라 1열→2열→3열로 바뀌는 카드(`card`) 레이아웃을 만들었습니다.
- `table table-bordered table-hover`로 기본 정보표 스타일을 빠르게 잡고, `badge`, `btn btn-outline-primary` 같은 Utility 성격의 컴포넌트로 강조 요소를 추가했습니다.
- 간격/정렬은 `g-4`, `gap-2`, `d-flex`, `flex-wrap`, `text-center` 같은 Utility Class로 처리했습니다.

## ⑤ 가장 어려웠던 점

Bootstrap grid와 직접 작성한 Flexbox(목표 섹션)를 한 페이지에 같이 쓰다 보니, 어떤 영역은 Bootstrap에 맡기고 어떤 영역은 직접 CSS로 만들지 기준을 잡는 게 헷갈렸습니다. 또 반응형으로 줄였을 때 목표 카드가 2열에서 1열로 바뀌는 시점(768px)을 정하고, 그 지점에서 padding·gap 값을 다시 조정하는 과정에서 화면을 계속 줄였다 늘렸다 하면서 눈으로 확인하는 데 시간이 걸렸습니다. Bootstrap의 class 우선순위와 제가 직접 쓴 selector 우선순위가 충돌하지 않는지 확인하는 것도 신경 쓰인 부분이었습니다.

## ⑥ AI는 어떻게 활용했나요?

작성한 HTML/CSS 구조를 AI에게 리뷰 요청해서 `header/nav/main/section/footer`가 적절히 쓰였는지, heading 단계가 자연스러운지, 표에 `scope` 속성이 빠지지 않았는지 같은 체크리스트 기준으로 점검받았습니다. AI에게 리뷰 요청해서 index파일의 표에 `<th scope="col">`을 넣어서 헤더 셀임을 명확히 했습니다. 또 mindset.html에서 `head` 태그 안에 `body`가 들어가 있던 구조 오류를 AI 리뷰를 통해 찾아냈고, 수정은 체크리스트와 비교해가며 반영했습니다. 반응형 breakpoint(768px, 480px)를 잡을 때도 AI에게 일반적인 기준을 물어본 뒤, 제 콘텐츠 양에 맞게 숫자를 조정하는 등 AI는 가능한 한 검토·점검 용도로 활용했습니다.
